自动回复质量评估流水线
背景团队上线了"客服自动回复"功能，需要评估回复质量。业务方的要求比较模糊（见 eval_criteria.md），你需要把它变成一套可运行的评估方案。
任务要求
1. 定义评估指标：从业务方的模糊描述中，定义至少 3 个可自动评估的指标，说明量化方式和选择理由。
2. 实现评估流水线：对 auto_replies.json 中 20 条自动回复逐条评分。可用真实 LLM API，也可用 mock 模式模拟返回。human_ref.json 提供了人工标注的参考回复和分析，可用来验证你的评估方法。
3. 讨论局限性：你的评估方法在哪些 case 上可能不准？怎么改进？
4. 输出评估报告：整体得分、各指标分布、最差 3 条 case 及分析。
5. README：简要说明指标定义、评估方法、局限性、AI 工具使用情况。
